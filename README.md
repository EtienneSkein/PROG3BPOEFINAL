Municify README

Overview:

The Municipal Service Application is designed to facilitate efficient management of community issues and events by providing users and administrators with tools to report and view local concerns. This application incorporates a variety of data structures to manage user data, service requests, and dependencies between tasks. The key features include submitting service issues, searching local events, tracking service request status, and generating issue reports.
This document serves as a comprehensive guide to understanding the Municipal Service Application. Below, you'll find a detailed explanation of how to compile, run, and clone the project from GitHub.

Project Setup and Compilation:
Prerequisites:
Visual Studio (recommended for C# development)
.NET Framework SDK
Git (for cloning the repository)

Cloning the Project from GitHub:
Clone the Repository:
First, ensure that Git is installed on your system.
Open a terminal (Command Prompt, PowerShell, or Git Bash).
Use the following command to clone the project repository:
git clone https://github.com/username/municipal-service-app.git
Navigate to the cloned directory: cd municipal-service-app
Open the Project
Launch Visual Studio.
Click on File > Open > Project/Solution.
Browse to the cloned directory and select the solution file (.sln).

How to Compile the Project:

Once the solution is open in Visual Studio, ensure that the build configuration is set to Debug or Release (as per your requirements).
Press F6 or click on Build > Build Solution.
If there are no errors, the solution should compile successfully, and the executable will be generated in the bin folder within the project directory.

Running the Project:

After a successful build, press F5 to run the project directly from Visual Studio.
The application window will open, and you can start using features such as Reporting Issues, Viewing Events, and Checking Service Request Status.
